This directory contains curl command samples for testing connectivity to various providers.

## Common Proxy Configuration

All providers support the following proxy configuration:

```bash
# HTTP/HTTPS Proxy
export HTTP_PROXY="http://proxy.example.com:8080"
export HTTPS_PROXY="http://proxy.example.com:8080"

# No proxy for specific hosts
export NO_PROXY="localhost,127.0.0.1,.internal.company.com"
```

## curl Proxy Options

```bash
# Use proxy
curl --proxy "${HTTPS_PROXY}" ...

# Skip TLS verification (testing only)
curl --proxy "${HTTPS_PROXY}" --insecure ...

# Use custom CA bundle
curl --proxy "${HTTPS_PROXY}" --cacert /path/to/ca-bundle.crt ...

# Use client certificate
curl --proxy "${HTTPS_PROXY}" --cert /path/to/client.crt --key /path/to/client.key ...
```

Each provider has specific environment variables. See individual files for details.

## Usage

1. Copy the environment variable exports to your shell or `.env` file
2. Replace placeholder values with actual credentials
3. Run the curl commands to test connectivity

## Troubleshooting

### Connection Timeout

```bash
# Add timeout options
curl --connect-timeout 10 --max-time 30 ...
```

### Verbose Output

```bash
# Show request/response details
curl -v ...

# Show only headers
curl -I ...
```

### SSL/TLS Issues

```bash
# Show SSL handshake details
curl -v --trace-ascii /dev/stdout ...

# Skip certificate verification (testing only!)
curl --insecure ...
```
